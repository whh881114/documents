# IELTS 网页维护规则

本文件适用于 `00-web/` 及其全部子目录，并与仓库根目录的 `AGENTS.md` 同时生效。若规则冲突，以本文件为准。

## 目录职责

- HTML 页面放在 `00-web/` 根目录；共享样式、页面行为和生成数据分别放在 `assets/css/`、`assets/js/`、`assets/data/`。
- 数据生成脚本放在 `scripts/`，生成文件必须注明来源，不得手工维护。
- `listening/`、`speaking/`、`reading/`、`writing/` 始终是只读数据源，网页任务不得修改其中任何文件。

## 页面与导航

- 听、说、读、写页面使用同一套页头、主导航、内容宽度、页脚和响应式规则。
- 新页面必须同步检查全站导航和主页入口，避免仍指向占位锚点。
- 列表页优先复用现有分类标签、列表行和状态样式；详情页优先复用标题区和正文排版。
- 所有链接使用仓库内相对路径，并正确处理中文、空格、撇号等字符。

## 内容与数据

- 不把 Markdown 正文手工复制到 HTML；由构建脚本读取资料并生成 `assets/data/` 下的数据文件。
- 修改资料目录结构或新增文章后，应重新运行对应脚本并同时更新生成结果。
- 自动生成文件只通过生成脚本更新。纯网页任务若发现原始资料问题，应报告但不得修改资料。
- 听力和阅读资料若在正文同目录存在 `review.md`，目录数据应记录复盘状态，列表显示“已复盘/未复盘”，详情页展示对应复盘内容。
- 渲染 Markdown 时先转义 HTML，只支持项目实际需要的有限语法。

## 样式与可用性

- 共用规则放在共享 CSS，模块专用规则放在对应模块 CSS；不在 HTML 中写内联样式。
- 不为单篇资料创建专用页面、脚本或样式。
- 页面需支持键盘操作和清晰焦点状态，手机宽度下不得横向溢出。
- 中文界面与英文原文统一使用 UTF-8，修改后检查乱码和特殊标点。

## 验证与交付

- 检查 HTML 引用的本地 CSS、JavaScript 和数据文件存在。
- 运行涉及的目录生成脚本，确认条目数量与资料目录一致，并抽查标题、来源、正文和图片路径。
- 检查主页及各模块导航、分类切换、详情返回链接、空数据和无效参数状态。
- 未经用户明确要求，不部署、不提交、不推送，也不修改学习资料目录。
