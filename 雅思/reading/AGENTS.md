# IELTS 阅读资料维护规则

本文件适用于 `reading/` 及其全部子目录。处理阅读学习资料前，应同时遵守仓库根目录的 `AGENTS.md`；若两者存在冲突，以本文件为准。

## 适用范围

- 只有用户明确要求新增、下载、整理或修正阅读学习资料时，才允许修改 `reading/`。
- 纯网页开发不得修改、移动、重命名或删除本目录中的学习资料。
- 不得提交 Cookie、账号凭据、登录会话、Token 或浏览器配置。
- 不修改 IELTS 文章原文的措辞、拼写和段落内容；只清理由网页界面混入的非正文内容。

## 话题分类目录

阅读真题按网站的 topic 分类存放，使用以下固定目录：

- `01-生物与环境/`（52 篇）
- `02-历史与发展/`（38 篇）
- `03-常识与科普/`（26 篇）
- `04-社会与文化/`（23 篇）
- `05-语言与教育/`（17 篇）
- `06-商业与经济/`（14 篇）
- `07-工业与建筑/`（10 篇）
- `08-医疗与健康/`（8 篇）
- `09-城市与国家/`（4 篇）

目录序号按当前文章数量从多到少排列。完成新的话题抓取或文章数量发生变化后，应重新核对数量；除非用户明确要求，不因少量增减自动重排目录。除非网站分类发生明确变化且用户要求同步，否则不要自行新增同义分类或调整文章所属分类。

## 文章目录和文件命名

每篇文章使用一个独立目录，正文固定保存为 `script.md`：

```text
reading/<两位序号>-<话题>/C<册号>-Test<测试号>-Passage<序号>-<英文标题>/script.md
```

示例：

```text
reading/02-历史与发展/C19-Test2-Passage1-The Industrial Revolution in Britain/script.md
```

命名时遵守以下规则：

- 册号、Test 和 Passage 编号从题目页面读取，不凭印象推断。
- 英文标题优先采用文章页面显示的正式标题。
- Windows 文件名不允许的字符 `< > : " / \\ | ? *` 替换为连字符 `-`，不要因此改写 `script.md` 内的标题。
- 去除目录名末尾的空格和句点。
- 不因大小写、标点或拼写观感而擅自修正原始标题。
- 若页面确实没有可提取的标题，应先人工核对，不能用 Questions 标题或页面按钮文字代替。

## `script.md` 内容格式

文件只保存阅读文章的标题和正文：

```markdown
# Article title

First paragraph...

Second paragraph...
```

具体要求：

- 第一行使用一级标题，内容为文章正式英文标题。
- 正文段落之间保留一个空行。
- 文件统一使用 UTF-8 编码，并以换行符结尾。
- 保留文章原有段落顺序和段落文字。
- 不把题目、答案、解析、用户作答状态或网页导航文字写入 `script.md`。
- 不在正文文件中额外添加来源说明、抓取日期或自动生成的元数据，除非用户明确要求。

## 文章图片

文章正文中的信息性图片是原文的一部分，不能因为正文以文字为主而省略。尤其是题目理解依赖的地图、示意图、流程图、结构图、照片或带图注的插图，必须与文章一起保存。

- 从文章正文区域识别图片，不下载网站 Logo、头像、按钮图标、收藏图标、答题控件、广告或其他界面装饰。
- 图片保存到对应文章目录中的 `images/` 子目录，不使用外部网址或本机临时路径作为最终引用。
- 优先保留网站提供的原始图片格式；文件名使用稳定、可读的名称，例如 `figure-01.png`、`map-01.jpg`。
- 在 `script.md` 中按图片在原文中的位置插入相对路径引用，例如 `![Figure 1](images/figure-01.png)`。
- 若网页提供图注或有意义的替代文本，应保留为 Markdown 图片说明；没有时使用中性的 `Figure 1`，不要虚构图片内容。
- 一篇文章包含多张信息性图片时，按页面出现顺序编号并全部保存。
- 图片下载后必须检查文件存在、大小非零、格式可识别，并确认 Markdown 相对路径有效。
- 若图片暂时无法下载，不得静默省略；应保留待处理说明并在交付时明确报告。
- 对之前已抓取的文章进行复查或发现正文图片时，应补回缺少的图片及 Markdown 引用，不因文章已经存在而跳过。

## 正文截取和噪音清理

从做题页面提取文章时，应只读取左侧文章区域，并在第一个题目标题之前停止。题目标题可能使用不同格式，至少应识别：

- `Question27-32`
- `Question 27–32`
- `Questions27-32`
- `Questions 27–32`

以下内容属于网页噪音，必须删除：

- `音频加载中，请稍后...`
- Questions 及其后面的所有题目、选项和答题说明
- 单独出现的题号数字
- 计时器、按钮、导航、收藏、报错等界面文字

不要使用只匹配 `Questions `（复数且带空格）的狭窄规则，以免 `Question27-32` 等形式导致题目混入文章。

## 浏览器抓取约定

- 用户打开具体题目标签页后，只读取题号、标题和左侧文章正文。
- 不选择答案，不提交试卷，不退出练习，不改变用户的作答内容。
- 批量处理时建议每组不超过 5 篇；每组完成后立即保存并校验，避免页面连接中断导致整批丢失。
- 已存在同名文章时先比较内容，不盲目覆盖；若文件已有用户修改，应保留并说明冲突。
- 完成抓取后释放浏览器连接，但不要关闭用户打开的标签页，除非用户明确要求。

## 完成后的核对

每批处理后至少检查：

1. 新增 `script.md` 数量与本批打开的文章数量一致。
2. 每篇目录中的册号、Test、Passage 和标题与页面一致。
3. 文件不包含 `音频加载中，请稍后...`。
4. 文件不包含以 `Question` 或 `Questions` 加题号开头的题目段落。
5. 文件不是异常空白或明显过短；若正文过短，应重新读取页面核对。
6. 中文路径、英文撇号、破折号和特殊字符没有乱码。
7. 完成一个话题后，将本地文章数量与网站该话题列表逐篇对照，列出遗漏项。
8. 检查正文区域是否含信息性图片；若有，确认本地图片数量、文件格式及 `script.md` 引用位置均正确。

## Git 约定

- 只提交本次阅读资料任务涉及的文件，不夹带其他未提交修改。
- 除非用户明确要求，不自动提交或推送。
- 提交前再次检查是否含题目、噪音、凭据或浏览器会话数据。
- 提交说明应明确指出新增或修正的是哪个阅读话题及文章范围。
